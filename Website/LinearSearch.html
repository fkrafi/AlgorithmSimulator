Linear Search

A linear search is the most basic of search algorithm you can have. A linear search sequentially moves through your collection (or data structure) looking for a matching value.

Example

For example, consider the following array, shown with array elements in sequence:

  • 7
  • 3
  • 6
  • 1
  • 0

Now, lets try to find 6

  • Iteration 1 (7 != 6):
    • 7
    • 3
    • 6
    • 1
    • 0
  • Iteration 2 (3 != 6):
    • 7
    • 3
    • 6
    • 1
    • 0
  • Iteration 3 (6 == 6):
    • 7
    • 3
    • 6
    • 1
    • 0

Now, lets try to find 5

  • Iteration 1 (7 != 5):
    • 7
    • 3
    • 6
    • 1
    • 0
  • Iteration 2 (3 != 5):
    • 7
    • 3
    • 6
    • 1
    • 0
  • Iteration 3 (6 != 5):
    • 7
    • 3
    • 6
    • 1
    • 0
  • Iteration 4 (1 != 5):
    • 7
    • 3
    • 6
    • 1
    • 0
  • Iteration 3 (0 != 5):
    • 7
    • 3
    • 6
    • 1
    • 0
  • Searched value is absent:
    • 7
    • 3
    • 6
    • 1
    • 0

Pseudo Code

LinearSearch(A[1..N], value) {
	for i ← 1 to N
		if A[i] == value then
			return i
	return not_found
}

Time Complexity

Binary search algorithm time complexity is O(log2(n)).

Simulator

Enter your numbers you want to search form in the "array" TextField area, separate each number with a comma(",").enter the number you want to search in the "search" textField area. After entering these numbers press the "start" button. You can put at most 25 numbers. Now the program is set search your number.

To see each step of sequential search one by one press the "next" button, to see the whole algorithm all at once press the "autoPlay" button so that the program will run the whole algorithm automatically.

If your desired number is found a pop up screen will appear saying "Found". If not another pop up screen will appear saying "Not Found".
