Selection Sort

Selection sort is the most conceptually simple of all the sorting algorithms. It works by selecting the smallest (or largest, if you want to sort from big to small) element of the array and placing it at the head of the array. Then the process is repeated for the remainder of the array; the next largest element is selected and put into the next slot, and so on down the line.

Example

For example, consider the following array, shown with array elements in sequence

  • 23
  • 78
  • 45
  • 8
  • 32
  • 56

The leftmost element is at index 1, and the rightmost element is at the highest array index, in our case, 6 (the effective size of our array is 6). The smallest element in this effective array (index 1-6) is at index 4. We have shown the smallest element and the one at the smallest index in red block with white text. We then swap the element at index 4 with that at index 1. The result is:

  • 8
  • 78
  • 45
  • 23
  • 32
  • 56

We reduce the effective size of the array to 5, making the smallest index in the effective array now 2. The smallest element in this effective array (index 2-6) is at index 4, so we swap elements at index 2 and 4 (red block with white text):

  • 8
  • 23
  • 45
  • 78
  • 32
  • 56

The smallest element in this effective array (index 3-6) is at index 5, so we swap elements at index 3 and 5 (red block with white text):

  • 8
  • 23
  • 32
  • 78
  • 45
  • 56

The smallest element in this effective array (index 4-6) is at index 5, so we swap elements at index 4 and 5 (red block with white text):

  • 8
  • 23
  • 32
  • 45
  • 78
  • 56

The smallest element in this effective array (index 5-6) is at index 6, so we swap elements at index 5 and 6 (red block with white text):

  • 8
  • 23
  • 32
  • 45
  • 56
  • 78

The entire array is now sorted.

Pseudo Code

for i ← 1 to n - 1
	k ← i
	for j ← i + 1 to n
		if A[j] < A[k] then 
			k ← j
	A[i] ↔ A[k]

Time Complexity

In general, we compare keys and move items (or exchange items) in a sorting algorithm (which uses key comparisons). So, to analyze a sorting algorithm we should count the number of key comparisons and the number of moves.

In selectionSort function, the outer for loop executes n-1 times. We invoke swap function once at each iteration.

The inner for loop executes the size of the unsorted part minus 1 (from 1 to n-1), and in each iteration we make one key comparison.

Therefore, numbers of key comparisons = 1+2+...+n-1 = n*(n-1)/2.

So, Selection sort is O(n2). The best case, the worst case, and the average case of the selection sort algorithm are same.
